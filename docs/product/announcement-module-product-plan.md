# 公告模块产品方案

## 1. 放在哪里

公告先放在左侧导航的「新闻资讯」里，第一版页面名叫「公告速递」。

原因很简单：公告属于信息流，但又比普通新闻更硬、更可追溯。它不适合塞进复盘页主体里，也不适合只放在数据总览里。复盘页以后只做“当天重点公告提醒”，完整列表和原文都放到「新闻资讯」。

第一版页面结构：

- 左侧导航：新闻资讯
- 页面标题：公告速递
- 主区域左侧：公告列表
- 主区域右侧：公告原文详情
- 顶部沿用全局日期选择器

## 2. 第一版先做什么

第一版只做一个能用的闭环：

1. 按日期看当天公告。
2. 每条公告显示股票代码、股票名称、公告类型、标题、公告日期。
3. 点击公告后，展示公告原文。
4. 原文获取后落到本地文件，后面重复打开优先读本地缓存。
5. 保留原始接口 JSON 和纯文本正文，不只保留摘要。

这一版不做：

- AI 解读
- 影响评分
- 公告和涨停、热门股的自动归因
- PDF 预览器
- 全市场公告搜索

这些后面加，不影响第一版上线。

## 3. 用户怎么看

### 看当天公告

用户进入「新闻资讯」后，默认使用顶部选中的交易日，页面展示这个日期的公告列表。

列表优先展示：

- 股票代码和名称
- 公告类型
- 公告标题
- 公告日期
- 是否已经缓存原文

### 看公告原文

点击列表里的公告，右侧展示原文。

展示顺序：

1. 标题
2. 股票信息
3. 公告类型和日期
4. 原文正文
5. 东方财富原网页链接
6. PDF 链接，如果接口提供

如果 PDF 下载被反爬拦住，不影响正文展示。正文和接口 JSON 先留住。

## 4. 后续怎么扩

### 第二步：筛选和搜索

增加公告类型筛选，例如：

- 业绩预告
- 股东大会
- 增减持
- 回购
- 重组
- 风险提示

同时加标题、代码、名称搜索。

### 第三步：个股关联

在个股卡片和热门股列表里增加「公告」入口。

用户点某个股票时，可以看到最近公告，而不是只能按日期翻。

### 第四步：公告解读

单独增加解读表，不覆盖原文。

解读内容包括：

- 一句话摘要
- 关键数字
- 可能利好、利空或中性
- 是否需要人工复核

### 第五步：盘前和复盘联动

在盘前指引里，只展示高影响公告。

在复盘首页里，只展示和当天市场表现有关的公告，例如：

- 热门股公告
- 涨停股公告
- 大额回购
- 业绩大幅变动
- 监管风险

## 5. 数据原则

底层数据必须完整留存：

- 公告列表原始数据留在数据库。
- 公告正文接口 JSON 留成本地 JSON 文件。
- 公告纯文本留成本地 TXT 文件。
- PDF 只有确认是有效 PDF 时才保存。
- AI 解读、人工备注、影响判断都另存，不改原文。

这样后面展示方式可以反复改，但底层数据不会丢。
