# 前端四模块重构方案

> 版本: v1.2 | 日期: 2026-07-03 | 状态: 已确认

---

## 一、背景与目标

### 现状问题

当前前端有 7 个平级 Tab，存在几个结构性问题：

1. **信息架构扁平**：7 个 Tab 地位等同，用户无明确路径，"复盘首页"和"涨停复盘"内容重叠
2. **模块边界模糊**：情绪数据分散在"量化全景""日内情绪""赚钱效应"三个 Tab 里
3. **资讯能力缺失**：新闻只在"盘前指引"里出现一次，没有独立的信息流
4. **社媒数据空白**：仅有热度排名，无论坛/社媒的实际内容和情绪分析

### 重构目标

将 7 Tab 整合为 **4 个一级模块**，每个模块有清晰的职责边界：

| 模块 | 定位 | 核心价值 |
|------|------|----------|
| **复盘** | 盘后分析主阵地 | 看懂今天发生了什么 |
| **盘中情绪** | 盘中实时监控 + 情绪周期 | 感知当前市场温度 |
| **新闻资讯** | 7×24 信息流 | 知道发生了什么事 |
| **社媒舆情** | 散户情绪 + 讨论热度 | 听到市场在说什么 |

## 二、信息架构

### 全局布局

左侧可折叠侧边栏 + 右侧内容区。侧边栏默认展开（图标+文字），可折叠为纯图标模式。

```
展开态 (200px)                         折叠态 (56px)
┌──────────┬───────────────────┐      ┌────┬───────────────────────┐
│  📊 复盘 │                   │      │📊  │                       │
│  🔥 情绪 │                   │      │🔥  │                       │
│  📰 资讯 │    模块内容区      │      │📰  │     模块内容区         │
│  💬 舆情 │                   │      │💬  │                       │
│          │                   │      │    │                       │
│ ──────── │                   │      │    │                       │
│  ⚙ 设置 │                   │      │⚙  │                       │
│  📋 任务 │                   │      │📋  │                       │
└──────────┴───────────────────┘      └────┴───────────────────────┘
     ▲                                        ▲
     └── 点击「◀」折叠                       └── 点击「▶」展开
```

**侧边栏交互：**
- 顶部：Logo + 折叠/展开按钮
- 中部：4 个一级模块（复盘、盘中情绪、新闻资讯、社媒舆情）
- 底部：辅助入口（数据总览、任务状态）
- 点击模块切换内容区，当前模块高亮
- 折叠态 hover 显示 tooltip（模块名）

-- 新闻持久化
CREATE TABLE news_daily (
    news_date text NOT NULL,
    source text NOT NULL,          -- cls, eastmoney, sina, cctv
    published_at text,
    title text NOT NULL,
    content text,
    url text,
    category text,                 -- policy, earnings, m&a, industry, overseas

    related_plates text,           -- JSON: ["白酒", "消费"]
    raw_payload text,
    created_at text NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY(news_date, source, title)
);

CREATE TABLE social_heat_daily (
    trade_date text NOT NULL,
    source text NOT NULL,          -- eastmoney_guba, ths_bar, xueqiu
    stock_code text NOT NULL,
    stock_name text,
    post_count integer,
    comment_count integer,
    view_count integer,
    sentiment_score real,          -- -1.0 ~ +1.0
    raw_payload text,
    created_at text NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY(trade_date, source, stock_code)
);
