## 3. 已完成的后端能力

新增统一复盘接口：

```text
GET /api/review/limit-up-reasons
GET /api/review/limit-up-tiers
GET /api/review/price-tiers
GET /api/review/promotions
GET /api/review/plate-rotation
GET /api/review/lhb
GET /api/review/movement-alerts
```

统一返回结构：

```text
date       当前交易日
status     ok / partial / empty / error
updated_at 生成时间
summary    汇总指标
filters    当前筛选条件
items      明细列表
warnings   数据缺失或降级说明
```

题材轮动接口额外返回：

```text
detail 当前选中题材详情
```

`detail` 内包含：

1. `trend`：近几天涨幅、评分、速度等趋势数据
2. `reasons`：题材入选原因、涨停家数、强度评分
3. `stocks`：题材代表个股、涨幅、换手、排名

## 5. 已修正的问题

### 5.1 涨幅梯队样本过少

旧逻辑要求股票必须同时拥有窗口第一天和最后一天 K 线，导致线上库只能算出 2 只股票。

新逻辑改为：

```text
每只股票在窗口内取自己的最早 K 线和最晚 K 线；
只要样本天数 >= 2，就参与涨幅梯队计算。
```

修正后，`2026-06-25` 的 10 日涨幅梯队可计算 67 只股票。

### 5.2 前端编码损伤

`ReviewWorkbench.tsx` 曾出现中文乱码和字符串损伤。本次已重写为干净 UTF-8 文件，并通过前端构建验证。

### 5.3 构建脚本写入权限

当前 Windows 环境不允许普通进程写入：

```text
web/node_modules/.tmp
web/node_modules/.vite-temp
```

所以构建脚本调整为：

```text
tsc -p tsconfig.app.json --noEmit
vite build --config vite.config.js --configLoader runner
```

这样可以避开 Vite 配置加载时写入 `node_modules` 临时目录。

## 9. 追加进展：晋级矩阵

本次补完“晋级”页。由于后端 `promotions` 接口返回的是按昨日高度分组的统计数据，而不是逐只股票明细，前端已经从旧的个股表格改成“昨日梯队 -> 今日结果”的矩阵卡片。

当前晋级页展示：

1. 昨日梯队：例如昨日 5 板、4 板、3 板。
2. 今日结果：晋级、保持、断板数量。
3. 晋级率：使用后端返回的 `advancement_rate`。
4. 结果条：用颜色快速区分晋级、保持、断板占比。
5. 断板名单：展示该梯队断板股票名称。
6. “只看晋级成功”开关：打开后只显示有晋级样本的昨日梯队。

这一步解决了之前页面和数据结构不匹配的问题：旧页面按个股字段展示，但接口实际返回的是梯队统计字段，导致前日高度、当日高度、股票名等字段无法正确显示。
