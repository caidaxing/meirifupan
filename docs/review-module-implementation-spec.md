# 复盘模块实施方案

本文只定义“复盘模块要做什么、每个页面需要什么数据、需要怎样的计算和展示”。本文不判断现有数据是否已经满足，也不因为暂时缺数据而缩小产品范围。数据可行性、已有/缺失对照、补数优先级，放到下一步单独评估。

## 1. 目标

把当前项目里的“涨停复盘”扩展成完整的“复盘模块”。复盘模块包含 7 个子页面：

1. 涨停原因
2. 涨停梯队
3. 涨幅梯队
4. 晋级
5. 题材轮动
6. 龙虎榜
7. 异动

这个模块的核心目标是：收盘后快速回答“今天市场主线在哪里、强度怎么样、接力是否顺畅、资金在哪里、风险有没有扩散、哪些方向值得明天继续观察”。

## 2. 页面总结构

### 2.1 复盘模块入口

前端增加一个一级页面或保持现有 `涨停复盘` Tab，但内部改成 7 个二级 Tab。

建议页面名称：

```text
复盘
```

二级 Tab：

```text
涨停原因 | 涨停梯队 | 涨幅梯队 | 晋级 | 题材轮动 | 龙虎榜 | 异动
```

### 2.2 公共页面元素

7 个子页面共用这些元素：

| 元素 | 内容 | 说明 |
| --- | --- | --- |
| 日期选择 | 当前交易日 | 所有子页都必须按交易日查看 |
| 复盘子导航 | 7 个子页面 | 子页之间切换不丢日期 |
| 数据更新时间 | 最近一次采集时间 | 避免用户误以为旧数据是当天数据 |
| 数据状态 | 正常、部分缺失、失败 | 只展示状态，不在本方案里判断来源 |
| 导出能力 | CSV、Markdown | 第一版可先做 CSV，Markdown 后补 |
| 股票点击 | 打开股票详情或右侧抽屉 | 后续统一股票详情入口 |

### 2.3 公共筛选能力

| 筛选 | 适用页面 | 说明 |
| --- | --- | --- |
| 题材 | 涨停原因、涨停梯队、题材轮动 | 按题材聚焦 |
| 连板高度 | 涨停原因、涨停梯队、晋级 | 看高标和接力 |
| 市场板块 | 涨幅梯队、龙虎榜、异动 | 看全市场表现 |
| 股票搜索 | 全部 | 输入代码或名称定位 |
| 是否涨停 | 涨幅梯队、龙虎榜、异动 | 区分涨停和非涨停强势股 |
| 是否异动 | 异动 | 只看已触发、即将触发、未触发 |

## 3. 子页面一：涨停原因

### 3.1 页面目标

涨停原因页面回答：

```text
今天哪些股票涨停？
它们分别属于什么题材？
每个题材有多少涨停？
每只股票为什么涨？
涨停质量怎么样？
```

### 3.2 页面结构

| 区域 | 内容 |
| --- | --- |
| 顶部统计 | 总涨停、首板、连板、最高板、炸板数 |
| 题材索引 | 题材名、涨停数量、强度分 |
| 题材分组列表 | 每个题材下面展示股票 |
| 股票表格 | 股票基础、涨停信息、资金和原因 |
| 原因展开 | 长文本涨停原因、事件催化、公告来源 |
| 实时行情开关 | 盘中可展示最新价、涨跌幅 |
| 导出 | 当前筛选结果导出 |

### 3.3 必要字段

#### 交易日字段

| 字段 | 说明 |
| --- | --- |
| `trade_date` | 交易日 |

#### 题材字段

| 字段 | 说明 |
| --- | --- |
| `plate_code` | 题材/板块代码 |
| `plate_name` | 题材/板块名称 |
| `plate_score` | 题材强度分 |
| `limit_up_count` | 题材内涨停数量 |
| `leader_stock_code` | 题材代表股代码 |
| `leader_stock_name` | 题材代表股名称 |

#### 股票字段

| 字段 | 说明 |
| --- | --- |
| `stock_code` | 股票代码 |
| `stock_name` | 股票名称 |
| `thscode` | 标准代码 |
| `limit_up_time` | 首次涨停时间 |
| `last_limit_up_time` | 最后涨停时间 |
| `board_count` | 连板高度 |
| `board_label` | 展示文案，如 `4连板`、`4天3板`、`首板` |
| `seal_amount` | 封单额 |
| `turnover` | 成交额 |
| `free_float_market_cap` | 实际流通市值 |
| `turnover_rate` | 实际换手率 |
| `reason` | 涨停原因长文本 |
| `concepts` | 题材标签列表 |
| `is_one_word_board` | 是否一字板 |
| `is_broken_once` | 是否炸开过 |
| `source` | 数据来源 |

#### 可选实时字段

| 字段 | 说明 |
| --- | --- |
| `last_price` | 最新价 |
| `change_pct` | 当前涨跌幅 |
| `updated_at` | 行情更新时间 |

### 3.4 计算逻辑

| 指标 | 计算方式 |
| --- | --- |
| 总涨停 | 当日涨停股票去重计数 |
| 首板 | `board_count = 1` |
| 连板 | `board_count >= 2` |
| 最高板 | `max(board_count)` |
| 题材涨停数 | 同一题材下股票去重计数 |
| 题材强度 | 外部强度分优先；没有时用涨停数、连板高度、封单、成交额合成 |
| 一字板 | 首次涨停时间接近开盘且全天未打开 |
| 炸开过 | 有涨停记录且盘中打开过 |

### 3.5 交互

| 交互 | 行为 |
| --- | --- |
| 点击题材索引 | 页面滚动到对应题材 |
| 点击股票 | 打开股票详情抽屉 |
| 展开原因 | 展示完整涨停原因 |
| 行情开关 | 增加最新价、涨跌幅列 |
| 搜索股票 | 定位股票并高亮 |
| 导出 | 导出当前筛选后的股票表 |

### 3.6 输出

这个页面要产出：

```text
今日涨停主线
每条主线的涨停数量
每条主线的核心股票
每只涨停股的上涨原因
涨停质量的初步判断
```

### 3.7 验收标准

1. 可以按日期查看涨停原因。
2. 可以按题材分组查看。
3. 每只股票至少有代码、名称、涨停时间、板高、题材、原因。
4. 题材内股票不能重复计数。
5. 导出结果与当前页面筛选一致。

## 4. 子页面二：涨停梯队

### 4.1 页面目标

涨停梯队页面回答：

```text
今天市场高度在哪里？
连板结构是否健康？
题材里谁是高标，谁是首板扩散？
炸板集中在哪些题材？
```

### 4.2 页面结构

| 区域 | 内容 |
| --- | --- |
| 顶部统计 | 最高板、连板数、首板数、炸板数 |
| 显示题材数量 | 5、8、10、12、15、20 |
| 题材横轴 | 强题材列表 |
| 梯队纵轴 | 最高板到首板、炸板 |
| 梯队格子 | 每个题材在每个高度里的股票 |
| 股票小卡 | 时间、名称、市场标识、当前涨幅 |
| 分时开关 | 展示关键股票分时小图 |

### 4.3 必要字段

| 字段 | 说明 |
| --- | --- |
| `trade_date` | 交易日 |
| `plate_code` | 题材代码 |
| `plate_name` | 题材名称 |
| `plate_score` | 题材强度分 |
| `stock_code` | 股票代码 |
| `stock_name` | 股票名称 |
| `board_count` | 连板高度 |
| `board_label` | 展示文案 |
| `limit_up_time` | 涨停时间 |
| `is_broken` | 是否炸板 |
| `market_tag` | 沪、深、创、科、北等 |
| `change_pct` | 当前或收盘涨跌幅 |
| `concepts` | 题材标签 |

### 4.4 计算逻辑

| 指标 | 计算方式 |
| --- | --- |
| 梯队层级 | 按 `board_count` 从高到低 |
| 首板 | `board_count = 1` |
| 炸板层 | 有涨停打开但收盘未封住 |
| 题材排序 | 题材强度分优先，其次涨停数量，再次最高板 |
| 股票排序 | 板高优先，其次涨停时间，最后封单额 |

### 4.5 交互

| 交互 | 行为 |
| --- | --- |
| 切换显示题材数量 | 控制横向题材数量 |
| 点击题材 | 过滤或跳转题材详情 |
| 点击股票 | 打开股票详情 |
| 开启分时 | 在股票卡片内展示小分时 |
| 切换是否显示炸板 | 显示或隐藏炸板层 |

### 4.6 输出

```text
最高板是谁
高标所属题材是什么
连板是否集中在主线题材
首板是否扩散
炸板是否集中
```

### 4.7 验收标准

1. 梯队从最高板到首板展示完整。
2. 炸板单独成层，不混入封住涨停。
3. 同一股票如果属于多个题材，可以在多个题材列出现，但统计总数时要去重。
4. 页面能清楚看出题材和高度的交叉关系。

## 5. 子页面三：涨幅梯队

### 5.1 页面目标

涨幅梯队页面回答：

```text
最近 10/20/30 天最强的股票是谁？
哪些股票没有涨停但趋势很强？
强势股主要集中在哪些题材？
哪些方向在涨停之外继续扩散？
```

### 5.2 页面结构

| 区域 | 内容 |
| --- | --- |
| 区间选择 | 10 天、20 天、30 天 |
| 起止日期 | 当前统计区间 |
| 上涨梯队 | 按涨幅区间分层 |
| 分组导航 | `100%+`、`80%-100%`、`60%-80%`、`40%-60%` 等 |
| 股票卡片 | 市场标识、名称、区间涨幅、题材 |
| 题材统计 | 每个题材在涨幅梯队里的股票数量 |
| 分享/导出 | 导出当前梯队 |

### 5.3 必要字段

| 字段 | 说明 |
| --- | --- |
| `end_date` | 统计截止日 |
| `start_date` | 统计开始日 |
| `window_days` | 统计窗口，如 10、20、30 |
| `stock_code` | 股票代码 |
| `stock_name` | 股票名称 |
| `thscode` | 标准代码 |
| `market_tag` | 沪、深、创、科、北 |
| `start_price` | 区间起始价 |
| `end_price` | 区间结束价 |
| `pct_change` | 区间涨幅 |
| `max_pct_change` | 区间最大涨幅，可选 |
| `concepts` | 题材标签 |
| `is_limit_up_today` | 当日是否涨停 |
| `limit_up_count_in_window` | 区间内涨停次数 |
| `turnover` | 区间成交额，可选 |

### 5.4 分层规则

第一版建议使用固定分层：

```text
>= 100%
80% - 100%
60% - 80%
40% - 60%
30% - 40%
20% - 30%
10% - 20%
0% - 10%
```

如果股票数量太多，页面默认只展开 `>= 40%`，其余折叠。

### 5.5 计算逻辑

| 指标 | 计算方式 |
| --- | --- |
| 区间涨幅 | `(end_price - start_price) / start_price * 100` |
| 起始价 | 统计窗口第一个交易日的收盘价 |
| 结束价 | 截止日收盘价 |
| 题材集中度 | 同一题材股票数量 / 当前梯队股票数量 |
| 涨停贡献 | 区间内涨停次数 |

### 5.6 交互

| 交互 | 行为 |
| --- | --- |
| 切换 10/20/30 天 | 重新计算梯队 |
| 点击分组 | 展开对应涨幅层 |
| 点击题材 | 筛选该题材股票 |
| 点击股票 | 打开股票详情和 K 线 |
| 只看涨停股 | 过滤区间内有涨停记录的股票 |
| 只看非涨停强势股 | 过滤未涨停但区间涨幅靠前的股票 |

### 5.7 输出

```text
阶段最强股票池
趋势强于涨停的股票池
题材扩散方向
补涨候选观察池
```

### 5.8 验收标准

1. 支持 10、20、30 天切换。
2. 每个区间都能显示起止日期。
3. 股票按涨幅从高到低排序。
4. 每只股票至少展示名称、代码、涨幅、题材。
5. 分层边界清楚，不能重复归入多个层级。

## 6. 子页面四：晋级

### 6.1 页面目标

晋级页面回答：

```text
昨天的连板股今天有多少继续涨停？
高标反馈是强还是弱？
接力资金是否愿意继续做高？
哪些题材晋级最好，哪些题材掉队最多？
```

### 6.2 页面结构

页面包含两个视图：

```text
晋级1：梯队板块
晋级2：连板详情
```

#### 晋级1：梯队板块

| 区域 | 内容 |
| --- | --- |
| 日期 | 显示基准日期，通常为前一交易日 |
| 显示题材数量 | 5、8、10、12、15、20 |
| 只看晋级 | 隐藏断板和失败反馈 |
| 晋级率 | 每个层级、每个题材的晋级比例 |
| 梯队矩阵 | 昨日梯队股票在今日的表现 |
| 分时 | 关键股票分时 |

#### 晋级2：连板详情

| 区域 | 内容 |
| --- | --- |
| 昨日连板池 | 昨日所有首板及以上股票 |
| 今日结果 | 晋级、断板、涨停失败、跌停、平盘等 |
| 个股表现 | 今日涨跌幅、最高涨幅、最低涨幅、成交额 |
| 反馈分类 | 强反馈、中性反馈、弱反馈 |

### 6.3 必要字段

| 字段 | 说明 |
| --- | --- |
| `base_date` | 基准日，通常是昨日 |
| `target_date` | 观察日，通常是今日 |
| `stock_code` | 股票代码 |
| `stock_name` | 股票名称 |
| `base_board_count` | 基准日连板高度 |
| `target_board_count` | 观察日连板高度 |
| `base_plate_names` | 基准日所属题材 |
| `target_plate_names` | 观察日所属题材 |
| `target_change_pct` | 观察日涨跌幅 |
| `target_high_pct` | 观察日最高涨幅 |
| `target_low_pct` | 观察日最低涨幅 |
| `is_promoted` | 是否晋级 |
| `is_limit_up` | 观察日是否涨停 |
| `is_broken` | 观察日是否炸板 |
| `is_limit_down` | 观察日是否跌停 |
| `feedback_type` | 反馈分类 |
| `limit_up_time` | 观察日涨停时间 |

### 6.4 晋级定义

| 情况 | 定义 |
| --- | --- |
| 首板晋级 | 昨日首板，今日再次涨停，变成 2 板 |
| 连板晋级 | 昨日 N 板，今日再次涨停，变成 N+1 板 |
| 断板 | 昨日涨停，今日未涨停 |
| 炸板失败 | 今日触及涨停但未封住 |
| 强反馈 | 未涨停但涨幅较高，或盘中冲高明显 |
| 弱反馈 | 大跌、跌停、深水 |

### 6.5 计算逻辑

| 指标 | 计算方式 |
| --- | --- |
| 总晋级率 | 晋级数量 / 基准日涨停数量 |
| 连板晋级率 | 昨日连板今日继续涨停数量 / 昨日连板数量 |
| 首板晋级率 | 昨日首板今日涨停数量 / 昨日首板数量 |
| 层级晋级率 | 某一板高晋级数量 / 该板高昨日数量 |
| 题材晋级率 | 某题材晋级数量 / 该题材昨日涨停数量 |
| 弱反馈比例 | 弱反馈数量 / 基准日涨停数量 |

### 6.6 交互

| 交互 | 行为 |
| --- | --- |
| 只看晋级 | 只保留 `is_promoted = true` |
| 点击层级 | 看该层级所有股票反馈 |
| 点击题材 | 看该题材所有股票反馈 |
| 点击股票 | 打开股票详情 |
| 切换详情视图 | 在矩阵和表格之间切换 |

### 6.7 输出

```text
今日接力是否顺畅
高标反馈是否正向
哪些题材晋级最好
哪些昨日强股今天亏钱效应大
```

### 6.8 验收标准

1. 明确显示基准日和观察日。
2. 晋级率分母必须来自基准日，而不是观察日。
3. 晋级、断板、炸板、弱反馈要分类清楚。
4. 支持按题材和板高查看。

## 7. 子页面五：题材轮动

### 7.1 页面目标

题材轮动页面回答：

```text
最近几个交易日主线怎么变化？
哪些题材持续走强？
哪些题材是新出现的？
哪些题材已经退潮？
```

### 7.2 页面结构

| 区域 | 内容 |
| --- | --- |
| 日期 | 当前观察日 |
| 窗口选择 | 最近 5、6、8、10、12、15 个交易日 |
| 排名列 | 每日题材强度排名 |
| 题材强度表 | 日期横向排列，显示题材名和分数 |
| 题材详情 | 点击题材后显示趋势、核心股、原因 |
| 二级题材 | 展示题材细分概念 |
| 图表切换 | 梯队、人气、涨幅、K 线 |

### 7.3 必要字段

#### 每日题材排名

| 字段 | 说明 |
| --- | --- |
| `trade_date` | 交易日 |
| `rank` | 当日排名 |
| `plate_code` | 题材代码 |
| `plate_name` | 题材名称 |
| `strength_score` | 强度分 |
| `limit_up_count` | 涨停数量 |
| `highest_board` | 最高板 |
| `turnover` | 成交额 |
| `change_pct` | 题材涨跌幅 |
| `leader_stock_code` | 领涨股代码 |

#### 题材详情

### 7.4 趋势阶段定义

| 强度变化 | 今日强度分 - 前一日强度分 |
| 延续性 | 活跃天数、排名稳定性、核心股反馈综合 |

## 9. 子页面七：异动

### 9.1 页面目标

异动页面回答：

```text
哪些股票已经触发交易所异动？
```

这里的“异动”先按交易所涨幅偏离异动来设计，不等同于普通盘中拉升、跳水提醒。盘中拉升/跳水可以后续放到“盘中观察”模块。

### 9.2 页面结构

| 异动表格 | 股票、累计涨幅、天数、今日触发、明日触发、后日触发、实时 |
| 状态标识 | 已触发、接近触发、未触发 |
| 股票风险提示 | 异动区间、可能停牌风险 |

| 字段 | 说明 |

### 9.4 规则设计

第一版需要支持两类规则：

| 规则 | 说明 |
| --- | --- |
| 10 日规则 | 连续 10 个交易日内涨幅偏离值累计达到阈值 |
| 30 日规则 | 连续 30 个交易日内涨幅偏离值累计达到阈值 |

不同市场、不同股票类型阈值可能不同，所以规则不能写死在页面里。后端需要输出每只股票适用的规则和阈值。

### 9.5 计算逻辑

| 指标 | 计算方式 |
| --- | --- |
| 累计涨幅 | 当前价相对窗口起点价的涨幅 |
| 偏离值 | 股票涨幅减去对应指数涨幅 |
| 今日触发价格 | 按规则反推今日达到阈值所需价格 |
| 明日触发价格 | 假设今日收盘价确定后，推算明日触发价 |
| 后日触发价格 | 继续向后滚动窗口推算 |
| 接近触发 | 距离阈值小于预设范围，例如 10% 内 |

### 9.6 交互

| 交互 | 行为 |
| --- | --- |
| 全部 | 展示所有候选 |
| 只要异动 | 只看已经触发或接近触发 |
| 排除异动 | 隐藏已触发股票 |
| 按实时涨幅排序 | 找最接近触发的股票 |
| 点击股票 | 查看触发计算详情 |

### 9.7 输出

```text
高位股异动风险
明日可能触发异动的股票
后日可能触发异动的股票
需要回避或重点观察的高位票
```

### 9.8 验收标准

1. 表格必须显示规则说明，避免误解。
2. 今日、明日、后日触发价格要分开。
3. 已触发和接近触发要有清楚状态。
4. 计算规则要能追溯，不做黑盒结论。

## 17. 判断标准

复盘模块做完后，用户打开一个交易日，应该能在 10 分钟内完成一次结构化复盘：

```text
1. 看涨停原因，确认主线。
2. 看涨停梯队，确认高度和结构。
3. 看涨幅梯队，确认趋势扩散。
4. 看晋级，确认接力反馈。
5. 看题材轮动，确认主线延续或切换。
6. 看龙虎榜，确认资金参与。
7. 看异动，确认高位风险。
```

这就是第一版复盘模块的完成标准。
